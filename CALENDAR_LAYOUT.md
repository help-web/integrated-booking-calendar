# 달력 레이아웃 명세

기존 구글 시트(2026년 운영본)의 월별 달력 구조를 웹앱에서 재현하기 위한 기준 문서다.
Univer 시트로 월별 달력을 그릴 때 이 레이아웃을 따른다. 추측하지 말고 이 문서대로 만든다.

## 전체 구조

- 한 달을 하나의 시트(탭)로 만든다. 1월~12월 탭이 있다.
- 한 주는 7개 열(월~일 순)로 가로 배치한다. 별도 요일 헤더 행은 두지 않는다. 요일은 날짜 라벨에만 표기한다(예. "1일 (목)").
- 한 주가 끝나면 다음 주가 아래로 이어진다. (한 달에 보통 5~6주)
- 한 날짜는 여러 행으로 쌓인 블록이다. 블록 통째 병합은 하지 않고, 행마다 따로 선택·입력한다. 열 테두리로 블록을 구분한다.
- 달력 셀의 세로 정렬 기본값은 위쪽(top)이다.
- 달력 맨 아래에 그 달의 일자별 아르바이트 출근표를 같은 시트에 이어 붙인다. (별도 화면으로 빼지 않는다.)

## 한 날짜 블록의 내부 순서 (위 → 아래)

1. 날짜 + 요일. (예. "16일 (월)")
2. 빈 회의실. 그날 예약 가능한 회의실 코드 목록. 파랑 계열 글씨. (예약 없을 때는 전체 회의실 목록을 기본 표시)
3. 마감 회의실. 그날 예약이 찬 회의실 코드 목록. 빨강 계열 글씨.
4. 예약 블록. 그날의 행사들. 행사가 여러 개면 세로로 쌓는다. 각 블록은 아래를 포함한다.
   - 고객사 / 담당자 (예. "▶ 한국기술교육대학교 / 담당자")
   - 계약 상태 (문의/견적/계약완료 등)
   - 사용 회의실 코드와 사용 시간
   - 날짜별 컨택 이력 (예. "[12.24] ...", "[12.26] ...")
5. 특이사항. 행사 외 메모(사전답사, 아르바이트 변경 공지, 주의 문구 등). 배경색으로 강조한다.

## 색상 규칙

기존 시트 기준의 기본값이다. 추후 사용자 피드백으로 조정한다.

- 빈 회의실. 파랑 계열 글씨.
- 마감 회의실. 빨강 계열 글씨.
- 토요일 날짜. 파랑 글씨. / 일요일 날짜. 빨강 글씨.
- 특이사항 강조. 연한 노랑·분홍·빨강 배경. (내용 성격에 따라 구분)
- 색상은 사람이 직접 칠하는 게 아니라, 회의실 상태·요일·메모 종류에서 자동으로 파생시킨다.

## 데이터 연결

- 빈/마감 회의실은 `booking_rooms`(active=true)를 날짜별로 집계해 계산한다. 전체 회의실 목록(`rooms`)에서 그날 예약된 회의실을 빼면 빈 회의실 목록이다. 예약된 회의실은 마감 행에 표시한다.
- 예약 블록은 `bookings` + `booking_rooms` 조인으로 가져온다. 시간은 `booking_rooms.start_time`/`end_time`, 회의실은 `room_id → rooms.code`다.
- 컨택 이력은 `contact_logs`를 예약별 날짜순으로 표시한다.
- 아르바이트 출근표는 `shifts` + `staff`를 그 달 날짜별로 집계한다.
- 2027년부터 운영하므로 초기에는 예약 데이터가 비어 있을 수 있다. 데이터가 없어도 빈 달력 골격은 정상적으로 그려져야 한다.

## 작업 순서 제안

1. 먼저 날짜 블록 골격(주 단위 7열 배치, 행별 편집, 테두리 구분)을 만든다.
2. 블록 안에 날짜·요일과 빈/마감 회의실 행을 채운다.
3. 예약 블록(행사)을 채운다.
4. 색상 규칙을 적용한다.
5. 맨 아래 아르바이트 출근표를 붙인다.

각 단계가 끝날 때마다 로컬에서 화면을 확인하고 다음으로 넘어간다. 한 번에 다 만들지 않는다.
